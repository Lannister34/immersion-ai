# Immersion AI

Современный интерфейс для ролевых чатов (RP) с локальными LLM-моделями. Лёгкий, быстрый и полностью автономный — все данные хранятся локально.

## Возможности

### Чат
- Потоковая генерация (SSE streaming) с возможностью остановки
- Редактирование и удаление любых сообщений
- Автоматическая генерация заголовков для чатов
- Системный промпт с шаблонами (EN/RU)
- Поддержка режима "размышлений" (thinking mode, Qwen3.5 и др.)
- Подсветка прямой речи и диалогов (русские кавычки «», тире)
- Markdown-форматирование в сообщениях
- Настройка сэмплера для каждого чата отдельно

### Персонажи
- PNG V2/V3 карточки персонажей (совместимы с SillyTavern / TavernAI)
- Создание, редактирование, удаление, импорт
- Drag & Drop / вставка аватаров (PNG, JPG, WebP → автоконвертация)
- AI-визард — пошаговый генератор персонажей (5 шагов: описание → имя → аватар-промпт → приветствие → сценарий)
- AI-генерация отдельных полей (имя, описание, первое сообщение, сценарий)

### Лорбуки (World Info)
- Создание и редактирование записей с ключевыми словами
- Автоматическая инъекция в контекст по совпадению ключей
- AI-генератор лорбуков по описанию мира

### Сценарии
- AI-генерация RP-сценариев (двухэтапная: краткий план → полный сценарий)
- Предпросмотр перед сохранением
- Управление списком сценариев

### LLM-сервер
- Встроенное управление llama-server / KoboldCpp (старт/стоп)
- Выбор GGUF-модели через файловый диалог
- Настройка GPU-слоёв, размера контекста, порта
- Режим внешнего сервера (подключение к уже запущенному KoboldCpp)
- Индикатор статуса подключения

### Настройки
- Параметры генерации: temperature, top_p, top_k, min_p, rep_pen и др.
- Пресеты сэмплера (Creative, Balanced, Precise, Deterministic, MinP)
- Имя пользователя для RP-контекста
- Переключение потокового режима

## Технологии

| Слой | Стек |
|------|------|
| **Frontend** | React 19, TypeScript, Vite 7, Tailwind CSS 4, Zustand 5, React Query 5 |
| **Backend** | TypeScript, Express, Sharp (обработка изображений) |
| **LLM** | llama-server / KoboldCpp (локальные GGUF-модели) |
| **Процессы** | PM2 (управление dev-серверами) |

## Требования

- **Node.js** >= 18
- **npm**
- **PM2** (глобально): `npm install -g pm2`
- **llama-server** или **KoboldCpp** для запуска моделей

## Установка и запуск

```bash
# 1. Клонируйте репозиторий
git clone <repo-url>
cd immersion-ai

# 2. Установите зависимости
npm run install:all

# 3. Запустите через PM2 (рекомендуется)
npm run dev
```

Откройте http://localhost:4778

### Ручной запуск (без PM2)

```bash
# Терминал 1 — бэкенд (порт 4777)
cd server && npm run dev

# Терминал 2 — фронтенд (порт 4778)
cd client && npm run dev
```

### Команды

| Команда | Описание |
|---------|----------|
| `npm run dev` | Запуск бэкенда и фронтенда через PM2 |
| `npm run stop` | Остановка всех PM2-процессов |
| `npm run restart` | Перезапуск |
| `npm run logs` | Логи PM2 |
| `npm run build` | Сборка клиента (production) |
| `npm run install:all` | Установка зависимостей сервера и клиента |

### Хранение данных

Все пользовательские данные хранятся в `data/` (gitignored):
- `data/characters/` — PNG-карточки персонажей
- `data/chats/` — чаты в формате JSONL
- `data/worlds/` — лорбуки (JSON)
- `data/scenarios/` — сценарии (JSON)
- `data/settings.json` — настройки подключения
- `data/user-settings.json` — пользовательские настройки
